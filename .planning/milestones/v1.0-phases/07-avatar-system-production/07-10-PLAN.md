<objective>
Close GAP-4: runtime PGRST204 crash when POST /avatars/me/reference-image is called.

The crash has two root causes that were fixed in the current session:
1. The `reference_image_url TEXT` column was never added to the `avatars` table in migration 004 — the backend wrote to a column that did not exist in the Supabase schema cache.
2. `AvatarResponse` in `avatar.py` did not include `reference_image_url` — even if the DB write succeeded, GET /avatars/me would never return the field, breaking the frontend polling loop.

Both code fixes are already applied on disk. This plan audits the fixes, commits them, and includes a mandatory checkpoint for the human to run the SQL in Supabase (the one step Claude cannot automate).

Purpose: Without this fix, every call to POST /avatars/me/reference-image crashes with HTTP 500 and the entire avatar onboarding flow is non-functional.
Output: Migration file and model file committed; Supabase schema updated via dashboard; VERIFICATION.md updated to record GAP-4 closed.
</objective>
