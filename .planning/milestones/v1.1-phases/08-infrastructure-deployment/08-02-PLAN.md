<objective>
Configure email DNS records (SPF, DKIM, DMARC) on the sending domain via Resend and update Supabase's Site URL to the production domain. Both are human-only steps that cannot be automated.

Purpose: Email DNS records take 24-48 hours to propagate globally. They must be added NOW — at the start of Phase 8 — so Phase 9 email features can start sending immediately without a waiting period (RESEARCH.md Pitfall 5). The Supabase Site URL update is required for Google OAuth in Phase 9.
Output: DNS records submitted at provider; Resend verification pending (expected — propagation takes time).
</objective>

<task type="checkpoint:human-action">
  <name>Task 1: Configure Resend sending domain and email DNS records</name>
  <action>
This is a human-only configuration task at two external dashboards. Claude cannot access DNS providers or the Resend Dashboard.

STEP 1 — Create Resend account (if not done):
  - Go to https://resend.com and sign up / log in
  - Navigate to Resend Dashboard → Domains → Add Domain
  - Enter your sending domain (e.g. yourdomain.com or mail.yourdomain.com)
  - Resend will display the exact DNS record values for your account — keep this page open

STEP 2 — Add DNS records at your DNS provider (Cloudflare, Namecheap, Hetzner DNS, etc.):
  Copy the exact values from the Resend Dashboard for each record:

  | Type | Host/Name              | Value (from Resend dashboard)                    |
  |------|------------------------|--------------------------------------------------|
  | TXT  | send                   | v=spf1 include:_spf.resend.com ~all              |
  | TXT  | resend._domainkey      | (public key from Resend — unique to your account)|
  | MX   | send                   | feedback-smtp.us-east-1.amazonses.com (priority 10) |
  | TXT  | _dmarc                 | v=DMARC1; p=none; rua=mailto:admin@yourdomain.com |

  NOTE: Use the exact values from the Resend Dashboard — the DKIM key is unique to your account and cannot be guessed. The SPF and DMARC values above are the standard Resend values but verify them in the dashboard.

STEP 3 — Click "Verify" in the Resend Dashboard:
  - Resend will check for the DNS records
  - It is normal and expected for verification to show "Pending" immediately — DNS propagation takes 2-48 hours
  - Leave the domain in "Pending" state — it will auto-verify once DNS propagates
  - You can check propagation progress with: dig TXT resend._domainkey.yourdomain.com

STEP 4 — Get Resend API key:
  - In Resend Dashboard → API Keys → Create API Key (full access)
  - Copy the key — it starts with re_
  - You will add this to backend/.env on the server as RESEND_API_KEY in Plan 03
  - Also note the "From" address format: "Ava <ava@yourdomain.com>"

STEP 5 — Update Supabase Site URL:
  - Go to Supabase Dashboard → your project → Authentication → URL Configuration
  - Change "Site URL" from http://localhost:3000 to https://yourdomain.com
  - Add https://yourdomain.com to the "Redirect URLs" allowlist (used by Google OAuth in Phase 9)
  - Save changes
  </action>
