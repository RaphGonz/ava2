<task type="checkpoint:human-action">
  <name>Task 1: Provision VPS, deploy application, and harden firewall</name>
  <action>
This is the production deployment. Follow these steps in order. Do not skip the SSH allow step before enabling UFW — that locks you out (RESEARCH.md Pitfall 1).

--- STEP 1: Provision VPS ---

  At Hetzner Cloud Console (https://console.hetzner.cloud):
  - Create new server: Ubuntu 24.04 LTS, CX32 (4 vCPU, 8 GB RAM)
    - CX22 has OOM risk during image generation bursts — use CX32 (STATE.md pending todo)
  - Add your SSH public key during creation
  - Note the server's public IP address

  At your DNS provider:
  - Add/update A record: yourdomain.com → [VPS public IP]
  - Wait 2-5 minutes for DNS to propagate before proceeding
  - Verify: dig +short A yourdomain.com (should return VPS IP)

--- STEP 2: Server initial setup ---

  SSH into the server:
  ```bash
  ssh root@yourdomain.com   # or: ssh root@[VPS IP]
  ```

  Run initial setup:
  ```bash
  apt update && apt upgrade -y
  apt install -y ufw fail2ban unattended-upgrades curl git
  # Install Node 20 (for frontend build)
  curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
  apt install -y nodejs
  # Install Docker
  curl -fsSL https://get.docker.com | bash
  # Verify
  docker --version && docker compose version
  ```

--- STEP 3: Clone repository ---

  ```bash
  mkdir -p /opt/ava2
  cd /opt
  git clone https://github.com/[your-repo-url]/ava2.git ava2
  # Or if private: git clone git@github.com:[your-org]/ava2.git ava2
  cd /opt/ava2
  ```

--- STEP 4: Edit Caddyfile with your real domain ---

  ```bash
  nano /opt/ava2/Caddyfile
  # Replace "yourdomain.com" with your actual domain
  # Save and exit (Ctrl+O, Enter, Ctrl+X)
  ```

--- STEP 5: Create backend/.env with production credentials ---

  ```bash
  nano /opt/ava2/backend/.env
  ```

  Add all of the following (replace all placeholder values with real credentials):

  ```bash
  # Supabase (Supabase Dashboard -> Settings -> API)
  SUPABASE_URL=https://<project>.supabase.co
  SUPABASE_ANON_KEY=eyJ...
  SUPABASE_SERVICE_ROLE_KEY=eyJ...

  # WhatsApp (Meta Developer Console -> App -> WhatsApp -> API Setup)
  WHATSAPP_ACCESS_TOKEN=...
  WHATSAPP_PHONE_NUMBER_ID=...
  WHATSAPP_VERIFY_TOKEN=...

  # OpenAI
  OPENAI_API_KEY=sk-...

  # Tavily
  TAVILY_API_KEY=tvly-...

  # ComfyUI Cloud
  COMFYUI_API_KEY=...

  # Stripe
  STRIPE_SECRET_KEY=sk_live_...
  STRIPE_WEBHOOK_SECRET=whsec_...
  STRIPE_PRICE_ID=price_...

  # Email (from Plan 02)
  RESEND_API_KEY=re_...
  RESEND_FROM_ADDRESS=Ava <ava@yourdomain.com>

  # App settings
  APP_ENV=production
  FRONTEND_URL=https://yourdomain.com

  # Redis (internal Docker network — do not change)
  REDIS_URL=redis://redis:6379
  ```

  CRITICAL: Set FRONTEND_URL to your actual domain — if left as default http://localhost:3000, CORS will block all API responses in production (RESEARCH.md Pitfall 6).

--- STEP 6: Create frontend/.env.production (for Google OAuth in Phase 9) ---

  ```bash
  cat > /opt/ava2/frontend/.env.production << 'EOF'
  VITE_SUPABASE_URL=https://<project>.supabase.co
  VITE_SUPABASE_ANON_KEY=eyJ...
  EOF
  ```

--- STEP 7: Run deploy.sh ---

  ```bash
  cd /opt/ava2
  chmod +x deploy.sh
  ./deploy.sh
  ```

  Expected output: 4 steps (build frontend, validate Caddyfile, restart services, health check).
  Watch for any errors. If Caddyfile validation fails, check the domain name syntax.

  Verify services are running:
  ```bash
  docker compose ps
  # All four services (caddy, backend, worker, redis) should show "running"
  ```

--- STEP 8: Verify HTTPS is working ---

  Wait 30-60 seconds for Caddy to obtain the TLS certificate (first-time only):
  ```bash
  docker compose logs caddy --tail=20
  # Look for: "certificate obtained" or "served with tls"
  ```

  Test from your local machine or any browser:
  - Visit https://yourdomain.com — app loads, no certificate warning
  - Run: curl -I http://yourdomain.com — must return 301 Moved Permanently with Location: https://yourdomain.com

--- STEP 9: Harden firewall with UFW ---

  IMPORTANT: Run these commands IN ORDER. Allow SSH first — never enable UFW before allowing 22.

  ```bash
  sudo ufw allow 22/tcp          # SSH — MUST be first
  sudo ufw allow 80/tcp          # HTTP (Caddy needs 80 for ACME challenge)
  sudo ufw allow 443/tcp         # HTTPS
  sudo ufw allow 443/udp         # HTTP/3 (QUIC)
  sudo ufw default deny incoming
  sudo ufw default allow outgoing
  sudo ufw enable                # Only after the allows above
  sudo ufw status verbose        # Verify — should show 22/80/443 ALLOW
  ```

  Also install fail2ban SSH protection (already installed in Step 2):
  ```bash
  systemctl enable fail2ban
  systemctl start fail2ban
  systemctl status fail2ban
  ```

--- STEP 10: Update WhatsApp webhook URL ---

  In Meta Developer Console → Your App → WhatsApp → Configuration:
  - Set Webhook URL to: https://yourdomain.com/webhook
  - Set Verify Token to: the value of WHATSAPP_VERIFY_TOKEN in your .env
  - Click "Verify and Save"
  - WhatsApp will call GET /webhook?hub.mode=subscribe&hub.verify_token=...&hub.challenge=... and verify the response
  </action>
