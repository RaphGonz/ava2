  <action>
**Pre-condition:** The human configuration steps below must be completed BEFORE running this task. Claude cannot access Supabase Dashboard or Google Cloud Console.

**Human actions required first:**

1. **Google Cloud Console** — Create OAuth 2.0 Client ID:
   - Go to: https://console.cloud.google.com -> APIs & Services -> Credentials -> Create Credentials -> OAuth 2.0 Client ID
   - Application type: Web application
   - Name: Avasecret
   - Authorized JavaScript Origins: `https://avasecret.org`
   - Authorized Redirect URIs: The Supabase callback URL (found in next step)
   - Save. Copy Client ID and Client Secret.

2. **Supabase Dashboard** — Enable Google provider:
   - Go to: Authentication -> Providers -> Google
   - Enable Google
   - Paste Client ID and Client Secret from step 1
   - Copy the callback URL shown (format: `https://<project-ref>.supabase.co/auth/v1/callback`)
   - Go back to Google Cloud Console and add this URL as an Authorized Redirect URI

3. **Supabase Dashboard** — URL Configuration:
   - Go to: Authentication -> URL Configuration
   - Site URL: `https://avasecret.org`
   - Redirect URLs: add `https://avasecret.org/**`

4. **Supabase Dashboard** — Send Email Hook:
   - Go to: Authentication -> Hooks -> Send Email
   - Hook type: HTTPS
   - URL: `https://avasecret.org/auth/send-email-hook`
   - Click "Generate secret" — copy the full secret (starts with `v1,whsec_`)

5. **Supabase Dashboard** — API Keys:
   - Go to: Settings -> API
   - Copy: Project URL (VITE_SUPABASE_URL) and anon/public key (VITE_SUPABASE_ANON_KEY)

**After human steps are complete — Claude actions:**

Add to `backend/.env` (append after existing vars):
```
SUPABASE_HOOK_SECRET=v1,whsec_<secret-from-supabase-hook-config>
```

Create or update `frontend/.env` with:
```
VITE_SUPABASE_URL=https://<project-ref>.supabase.co
VITE_SUPABASE_ANON_KEY=<anon-public-key-from-supabase>
```

Note: `frontend/.env` may already exist — check before creating. These are the env vars noted in STATE.md as required for v1.1.

**Redeploy to production:**

SSH to VPS and run deploy script to pull the new code and rebuild:

```bash
# On the VPS (SSH in first, then):
cd /opt/ava2
git pull origin main
./deploy.sh
```

The deploy script rebuilds the frontend (which bakes VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY into the JS bundle) and restarts the backend (which reads SUPABASE_HOOK_SECRET from .env).

**Verify backend started correctly with new config:**
```bash
# On VPS:
docker compose logs backend --tail=20 | grep -E "ERROR|supabase_hook|resend"
```
