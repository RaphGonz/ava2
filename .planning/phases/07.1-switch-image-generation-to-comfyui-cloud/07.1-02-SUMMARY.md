## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] ComfyUI _poll_and_download passed status response to _download_output instead of history data**
- **Found during:** Task 2 verification (full test suite run)
- **Issue:** `_poll_and_download` called `_download_output(client, data, output_node)` where `data` was the status response `{"status": "completed"}` — no outputs field. The actual output node filenames live in `/api/history_v2/{id}`. This caused `RuntimeError: ComfyUI: no images found in output node '60'` in all image generation tests.
- **Fix:** Added `_fetch_history_and_download` method that calls `/api/history_v2/{prompt_id}`, retries up to 3 times on empty outputs (handles race condition), then passes history data to `_download_output`. `_poll_and_download` now calls this method after status=completed.
- **Files modified:** `backend/app/services/image/comfyui_provider.py`
- **Verification:** 5/5 test_comfyui_provider tests pass; 52/52 full suite passes
- **Committed in:** `8e30125` (Task 2 commit)

**2. [Rule 1 - Bug] Seed not randomized in _build_t2i — same seed returned on every call**
- **Found during:** Task 2 verification (test_seed_is_randomized_across_calls failure)
- **Issue:** `_build_t2i` loaded the workflow JSON and set the prompt but did not overwrite the seed value — every call produced the same seed (203495258957698 from the JSON file), resulting in identical images.
- **Fix:** Added `import random` and `workflow["86:3"]["inputs"]["seed"] = random.randint(0, 2**32 - 1)` in `_build_t2i`. Linter also applied the same pattern to `_build_i2i` (node `89:65`).
- **Files modified:** `backend/app/services/image/comfyui_provider.py`
- **Verification:** test_seed_is_randomized_across_calls passes with different seeds across calls
- **Committed in:** `8e30125` (Task 2 commit)
