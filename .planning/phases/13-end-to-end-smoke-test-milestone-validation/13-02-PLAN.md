  <action>
    Execute the automated portion of the smoke test runbook.

    Step 1 — Pre-flight health check:
    ```bash
    curl -sf https://avasecret.org/health && echo "PASS: server alive" || echo "FAIL: server unreachable"
    ```
    Expected: `{"status":"ok"}` — if FAIL, stop and report: production server is not reachable.

    Ask user to run on VPS: `docker compose ps` — `worker` service must show status `running`.

    Step 4 — Obtain production JWTs:
    - SMOKE_UNSUBSCRIBED_JWT: Ask user to log in to https://avasecret.org as an account with no active subscription, then open DevTools > Application > Local Storage > find key matching `sb-*-auth-token` > copy the `access_token` value
    - SMOKE_REGULAR_USER_JWT: Ask user to log in as any non-admin production user and copy the access_token the same way

    Step 5 — Run paywall and admin access control tests:
    ```bash
    cd backend
    SMOKE_UNSUBSCRIBED_JWT="<paste>" SMOKE_REGULAR_USER_JWT="<paste>" python -m pytest tests/test_smoke_paywall.py -x -v
    ```
    All 3 tests must PASS. Record results.

    Step 6 — Run usage_events coverage test:
    ```bash
    cd backend && python -m pytest tests/test_smoke_usage_events.py -x -v
    ```
    Must PASS (not SKIP). If SKIP: production DB env vars not configured locally — ask user to run from VPS directly or temporarily set SUPABASE_URL + SUPABASE_SERVICE_KEY locally to production values.

    Step 7 — Update the evidence table in docs/smoke-test-runbook.md with automated test results:
